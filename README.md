# Гайд по развертыванию приложения
# *Данный гайд создан на основании Debian дистрибутива (может быть совместим с Ubuntu, Linux Mint и на других дистрибутивах, которые основаны на Debian)*
Данный проект написан на фреймворке Django, использует базу данных PostgreSQL в купе с gunicorn сервером и реверс-прокси nginx сервером. Гайд построен по пошаговой структуре, поэтому просто следуйте нумерации.
## Первоначальные требования
Должны быть предустановлены:
- Python 3.10+
- PostgreSQL 14+
- Git
- Nginx

##  Клонирование репозитория
Для начала, нужно клонировать репозиторий. Откройте терминал и вставьте следующую команду:
```bash
git clone https://github.com/CasinoRoyal666/dtst777.git
```
После заверешения клонирования, перейдите в папку репозитория
```bash
cd dtst777/
```

## Создание и настройка виртуального окружения
Находясь в папке репозитория, в терминале выполните:
```bash
python3 -m venv .venv
```
(вы можете назвать окружение по другому, к примеру просто venv)
Далее, активируйте виртуальное окружение
```bash
source .venv/bin/activate
```
После чего установите зависимости, список которых находится в файле requirements.txt
```bash
pip install -r requirements.txt
```

## Настройка переменных окружения
В целях конфиденциальности данных в проекте представлен пример файла с переменными окржужения под названием `.env.example`,вам нужно либо переименовать его, либо создать новый файл с именем `.env` на том же уровне что и `.env.example`. В командах ниже рассмотрен второй вариант.
```bash
cp .env.example .env
```

Откройте терминальный клиент Postgre, введя в терминале команду:
```bash
sudo -u postgres psql
```
Далее создайте базу, пользователя и его привилегии
```sql
CREATE DATABASE имя_базы;
CREATE USER пользователь WITH PASSWORD 'пароль';
ALTER ROLE пользователь SET client_encoding TO 'UTF8';
ALTER ROLE пользователь SET default_transaction_isolation TO 'read committed';
ALTER ROLE пользователь SET timezone TO 'UTC';
GRANT ALL PRIVILEGES ON DATABASE имя_базы TO пользователь;
GRANT ALL ON SCHEMA public TO пользователь;
ALTER DATABASE имя_базы OWNER TO пользователь;
```
после чего выйдите из терминального клиента
```sql
\q
```
После этого, нужно сгенерировать секретный ключ Django, находясь в корневой директории репозитория, командой, показанной ниже:
```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

## 5. МИграции и статика
Нужно создать и применить миграцию для появления сущностей в базе данных и дальнейшей работе с ними. Управление django проектом происходит через `manage.py`, в нем есть две команды - `makemigrations` для того чтобы автоматически сгенерировать миграцию основанную на моделя, а также - `migrate` - для применения этой миграции к базе данных. Однако создавать миграции пригодится в дальнейшем, при добавления вами функционала в проект. Миграция идет вместе с репозиторием, так что вам нужно только ее применить к своей базе данных.
Все еще находясь в корне репозитория (на уровне `manage.py`) выполните следующую комманду:
```bash
python manage.py migrate
```
Также, в `manage.py` есть комманда которая собирает статические файлы (CSS, JS) в одну директорию, указанную пользователем. Директория указана в файле `core/settings.py` (STATIC_ROOT, строка 124), вы можете изменить ее по желанию. Выполните команду:
```bash
python manage.py collectstatic
```

## 6. Настройка gunicorn
Gunicorn - HTTP-сервер, предназначенный для запуска Python приложений. Далее, будет описан процесс его настройки и запуска
**Повторяю, данный гайд подарзумевает использование Debian, либо Debian-based дистрибутивов!**
В терминале нужно создать systemd-юнит, командой ниже:
```bash
sudo nano /etc/systemd/system/gunicorn.service
```
nano создает новый файл и открывает текстовый редактор, в котором нужно написать конфиг для Gunicorn. Пример конфига представлен ниже:
```ini
[Unit]
Description=Gunicorn daemon
After=network.target

[Service]
User=твой-пользователь
Group=www-data
WorkingDirectory=/путь/до/проекта
ExecStart=/путь/до/проекта/.venv/bin/gunicorn \
          --workers 3 \
          --bind unix:/путь/до/проекта/gunicorn.sock \
          core.wsgi:application

[Install]
WantedBy=multi-user.target
```
`CTRL + X` для сохранения и `CTRL + O` для выхода из файла
После создания конфига, запускаем Gunicorn и включаем его автозапуск посредством нижепреведенных команд
```bash
sudo systemctl daemon-reload
sudo systemctl start gunicorn
sudo systemctl enable gunicorn
```
После чего нужно проверить статус сервера
```bash
sudo systemctl status gunicorn
```
Индикатором рабоспособности является статут `Active: active (running)`, после чего `CTRL + C` для закрытия статуса.

## Настройка Nginx
Добавляем проект в Nginx с конфигом, для начала создаем файл конфига:
```bash
sudo nano /etc/nginx/sites-available/проект
```
После чего, в открывшемся редакторе пишем конфиг. Его пример представлен ниже
```nginx
server {
    listen 80;
    server_name твой-домен-или-ip (можно просто 127.0.0.1 для локального развертнывания);

    location = /favicon.ico { access_log off; log_not_found off; }

    location /static/ {
        root /путь/до/проекта;
    }

    location / {
        include proxy_params;
        proxy_pass http://unix:/путь/до/проекта/gunicorn.sock;
    }
}
```
`CTRL + X` для сохранения и  `CTRL + O` для выхода, после чего нужно активировать написанный конфиг командами ниже:
```bash
sudo ln -s /etc/nginx/sites-available/проект /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 8. Права доступа
Для корректной работы нужно выдать Nginx доступ к домашней папке пользователя. Это делается командами ниже:
```bash
sudo chmod o+x /home/твой-пользователь
sudo usermod -aG www-data твой-пользователь
``` 
После чего перезапустить Gunicorn сервер:
```bash
sudo systemctl restart gunicorn
```
